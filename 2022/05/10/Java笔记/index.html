2.变量的赋值

如果变量是基本数据类型，此时赋值的是变量所保存的数据值。

如果变量是引用数据类型，此时保存的是变量所保存的数据的地址值。

3.方法的形参传递机制

  1. 形参：方法定义时，声明在小括号内的参数。
  2. 实参：方法调用时，给形参传递的数据。
  3. 值传递机制：如果参数时基本数据类型，此时实参赋给形参的是实参真实存储的数据值。

4.封装与隐藏

  1. 封装性的引入
    1. 当我们创建一个类的对象以后，我们可以通过”对象.属性”的方式，对对象的属性进行赋值，这里赋值操作要受到属性的数据类型和存储范围的制约除此之外，没有别的制约条件。
    2. 实际问题中我们往往需要给属性赋值加入额外的限制条件，但是这个条件不能在属性声明时体现，我们只能通过方法来进行条件的限制，同时我们要避免用户再使用“对象.属性”的方式进行赋值，我们需要将属性声明为私有的（private），需要提供关于属性的方法来进行调用。
  2. 封装性的体现
    1. 不对外暴露的私有的方法
    2. 四种权限修饰的理解，Java规定的四种权限修饰符：（从小到大）： private ，缺省（default），protected，public
      对于class的权限修饰只可以使用 public 和 default。
    3. public类可以在任意地方被访问
      default类只可以被同一个包内部的类访问
    4. 四种权限修饰符可以用来修饰类及累的内部结构：属性，方法，构造器，内部类
  3. 构造器的作用

​ 1.创建对象，定义构造器的格式：权限修饰符 + 类名（形参列表）{},可以用来初始化对象信息。

​ 2.类的构造器中，可以使用this（形参列表）的方式来调用其他构造器，不能调用自己，this（形参列表）必须声明当前构造器的 首行。

​ 3.内部只能声明一个构造器来调用其他构造器

5.继承性

1.继承性减少了代码的冗余，提高了代码的复用性，便于功能的拓展，为之后多态性的使用提供前提。

2.继承性格式class A extends B{ } ，A子类 B父类，继承父类中的所有属性和方法，子类继承父类之后可以声明自己特有的方法和 属性

6.方法的重写

  1. 子类继承父类以后，可以对父类中的同名参数的方法，进行覆盖操作
  2. 方法的声明：权限修饰符 + 返回值类型 + 方法名（形参列表）
  3. 子类重写方法的权限修饰符不小于父类被重写的权限修饰符
  4. 特殊情况：子类不能重写父类中声明为private权限的方法
  5. 返回值类型可以是父类的子类。如果是基本数据类型，则子类重写必须是相同的数据类型
  6. 属性不会覆盖。super调用父类中的属性

7.super的使用

1.我们可以在子类的方法或构造器中。通过使用“super.属性”或“super.方法”的方式,显示调用父类声明的属性方法。

 2.在子类中调用指定父类中的构造器采用“super（构造器）”的方式，这种方式的使用必须声明在子类构造器首行！
 3. this（形参列表） 和 super（形参列表）只能二选一
 4. 若首行没有声明，则默认调用父类中空参构造器super（）

8.子类对象实例化全过程

1.从结果上看：子类继承了父类以后，就获取了父类中声明的属性或方法。创建子类的对象就会在堆空间中加载所有父类中声明的属性。

2.从过程上看，当我们创建子类的构造器创建子类对象时，我们一定会直接间接调用父类构造器，进而调用父类的构造器。

9.多态

1.多态指的是父类的引用指向子类的对象。

2.多态的使用：当调用子父类同名同参数的方法时，实际执行的是子类重写父类的方法，——>虚拟方法的调用

3.有了对象的多态性以后，在编译期只能调用父类中声明的方法，运行期，我们实际执行的是子类重写父类的方法。

4.使用前提：类的继承关系，方法的重写。

5.多态性只适合方法不适合属性，是运行时行为
